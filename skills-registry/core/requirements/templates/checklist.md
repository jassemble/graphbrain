# Requirements Review Checklist

## Completeness
- [ ] Problem statement is specific (not "improve X")
- [ ] Target users identified with concrete workflows
- [ ] Success metrics are numeric and measurable
- [ ] All acceptance criteria are machine-verifiable boolean
- [ ] Error/failure cases covered (not just happy path)
- [ ] Out of scope section exists and is non-empty
- [ ] Dependencies listed with owners and timelines

## Quality
- [ ] No solution prescribed in requirements (WHAT, not HOW)
- [ ] No subjective language ("should", "might", "could", "good")
- [ ] Each criterion independently testable by a machine
- [ ] Performance thresholds specified where applicable

## Scope
- [ ] Completable in 1-3 days per sub-feature
- [ ] No more than 10 acceptance criteria per PRD
- [ ] No hidden dependencies on unbuilt features
- [ ] Stakeholders identified and priorities documented
